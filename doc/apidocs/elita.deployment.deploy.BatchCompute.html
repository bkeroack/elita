e.d.d.BatchCompute : class documentation

Part of elita.deployment.deploy View In Hierarchy

Given a list of application groups that require rolling deployment and an (optional) list that do not, compute the optimal batches of server/gitdeploy pairs. All non-rolling groups are added to the first batch. Splitting algorithm is tolerant of outrageously large rolling_divisors.

Written in a functional style to facilitate testing.

Combine the big list of batches into a single list.

Function is passed a list of lists:
[
    [ { "servers": [...], "gitdeploys": [...] }, ... ],     # batches 0-n for group A
    [ { "servers": [...], "gitdeploys": [...] }, ... ],     # batches 0-n for broup B
    ...
]

Each nested list represents the computed batches for an individual group. All nested lists are expected to be
the same length.
